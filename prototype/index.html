Design Decision: Two-source tag system.
The current SKILL.md frontmatter only has name, description, version. This prototype proposes:
1. Auto tags (purple) — from a new tags field in SKILL.md frontmatter. Read-only for remote skills.
2. Custom tags (cyan) — user-added via the detail panel. Stored locally, won't modify the skill file.
Both types appear as filter tabs. Users can organize any skill (even remote) into custom groups like "favorites" or "my-team".
